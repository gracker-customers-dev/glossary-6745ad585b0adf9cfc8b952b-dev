Single sign-on

Single sign-on (SSO) is an authentication process that allows a user to access multiple applications with one set of login credentials.

Description

Single sign-on (SSO) is a user authentication process that enables a user to log in once and gain access to multiple applications or systems without needing to log in again for each one. In the context of governance frameworks, SSO enhances security and user experience by minimizing the number of credentials that users must remember and manage. This can lead to reduced password fatigue and lower the risk of password-related security breaches. SSO systems typically involve a centralized authentication server that validates user credentials and issues tokens or assertions to allow access to various services. SSO can be implemented in various environments, including corporate intranets, cloud services, and mobile applications. Governance frameworks often include policies and procedures for managing user identities, access controls, and compliance requirements, making SSO a critical component to ensure secure and efficient access management across an organization’s resources.

Examples

  • Google Workspace allows users to access Gmail, Drive, and other services with one Google account.
  • Microsoft Azure Active Directory provides SSO capabilities for numerous enterprise applications.

Additional Information

  • SSO can improve user productivity by reducing the time spent logging into multiple services.
  • Implementing SSO requires careful consideration of security measures, such as multi-factor authentication.
